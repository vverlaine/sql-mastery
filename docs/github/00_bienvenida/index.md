# Bienvenido al Curso de GitHub

**De analista que guarda archivos a analista que colabora en código.**

Hasta ahora, en los pilares anteriores aprendiste a escribir SQL, transformar datos con Spark y organizar proyectos profesionales en VSCode + Databricks. Pero hay un pedazo crítico que falta: **cómo guardas, versionas y compartes ese código con tu equipo**.

Este pilar resuelve exactamente eso. Vas a aprender Git y GitHub: las herramientas que la industria entera usa para versionar código, colaborar entre múltiples personas, recuperar versiones anteriores cuando algo se rompe, y construir un historial completo de todas las decisiones del proyecto.

---

## Por qué este pilar es importante

Imagínate este escenario real: estás trabajando en un análisis crítico para CBC. Tu colega te pide ayuda con su parte del proyecto, así que le envías tu archivo `analisis.py` por Slack. Él hace cambios y te lo manda de vuelta. Mientras tanto, tú también lo modificaste. Ahora tienen dos versiones distintas del mismo archivo y nadie sabe cuál es "la buena".

Ese problema es el problema que Git resuelve. Y lo resuelve de forma elegante, con herramientas pensadas hace 20 años por los ingenieros que construyen Linux. Si dominas Git, dominas el lenguaje universal de la colaboración técnica moderna.

---

## ¿Qué aprenderás?

Al finalizar este curso serás capaz de:

- ✅ **Entender los conceptos fundamentales** de Git: repositorio, commit, branch, merge
- ✅ **Versionar tu trabajo localmente** con confianza
- ✅ **Trabajar con branches** para experimentar sin romper lo que ya funciona
- ✅ **Conectar tu trabajo con GitHub Enterprise** de CBC
- ✅ **Colaborar con tu equipo** mediante Pull Requests y code reviews
- ✅ **Resolver conflictos** cuando dos personas modifican el mismo código
- ✅ **Integrar GitHub con Databricks Repos** para tener tu código siempre sincronizado
- ✅ **Aplicar el workflow profesional** del día a día en CBC

---

## ¿Para quién es este curso?

Para analistas que ya completaron los módulos anteriores y quieren cerrar el círculo de su workflow profesional. Si ya estructuras proyectos en VSCode y trabajas con Databricks, este pilar es la pieza que conecta todo: tu código deja de vivir aislado en tu computadora y empieza a vivir en el ecosistema colaborativo del equipo.

No necesitas experiencia previa con Git. De hecho, este curso asume que NUNCA has hecho un commit en tu vida.

---

## Cómo vamos a aprender

Hay dos formas de usar Git: **línea de comandos (CLI)** y **interfaz visual** (en este caso, el panel de Source Control de VSCode). En este curso vamos a aprender ambas en paralelo.

¿Por qué? Porque cada una tiene sus ventajas:

| Línea de comandos | Interfaz de VSCode |
|---|---|
| Más rápida una vez la dominas | Más visual al inicio |
| Funciona en cualquier sistema | Solo dentro de VSCode |
| Indispensable cuando algo sale mal | Cómoda para flujos cotidianos |
| Es como hablan los desarrolladores | Reduce la curva de aprendizaje |

> 💡 **Mi recomendación honesta:** Aprende ambas. Empieza con la interfaz visual cuando estés cansado o tengas prisa, y usa la línea de comandos cuando quieras entender REALMENTE qué está pasando o cuando necesites resolver un problema complejo.

Cada lección del curso muestra los dos modos. Tú decides cuál usar en cada momento.
